# 실서비스 전환 계획

프롬포트랩을 공개 데모에서 실제 커뮤니티로 바꿀 때 필요한 작업 순서입니다.

## 1. 지금 먼저 막아야 할 것

- 임시 `localStorage` 로그인은 실제 인증이 아니므로 실서비스 로그인처럼 보이면 안 됩니다.
- 글쓰기/댓글 버튼은 DB 저장이 붙기 전까지 실제 제출처럼 보이면 안 됩니다.
- 사용자가 올린 이미지를 저장하기 전에는 용량 제한, 파일 형식 제한, 삭제 정책이 필요합니다.

## 2. Supabase에서 만들 테이블

- `profiles`: 사용자 공개 프로필
- `image_posts`: 이미지 프롬프트 게시글
- `prompt_versions`: 한 이미지에 연결되는 한국어/영어/네거티브 프롬프트 원문
- `board_posts`: 글 중심 게시판 게시글
- `comments`: 이미지 글과 일반 글 공용 댓글
- `reactions`: 좋아요, 저장 등 사용자 반응
- `reports`: 신고 내역

## 3. 권한 원칙

- 누구나 공개 글과 댓글을 읽을 수 있습니다.
- 로그인한 사용자만 글, 댓글, 반응, 신고를 만들 수 있습니다.
- 작성자만 자기 글과 댓글을 수정/삭제할 수 있습니다.
- 관리자는 별도 `role` 값으로 삭제/숨김 처리를 할 수 있게 둡니다.
- DB 보안은 클라이언트 코드가 아니라 Supabase RLS 정책으로 막습니다.

## 4. 이미지 업로드 원칙

- 허용 확장자: `jpg`, `jpeg`, `png`, `webp`
- 초기 최대 용량: 5MB
- 업로드 경로: `image-posts/{user_id}/{post_id}/{filename}`
- 원본 이미지를 공개할지, 썸네일을 별도로 만들지는 다음 단계에서 결정합니다.

## 5. 다음 구현 순서

1. Supabase 프로젝트 생성
2. `.env.local`에 Supabase URL/Anon Key 입력
3. `supabase/schema.sql` 실행
4. Supabase Auth 연결
5. 글쓰기/댓글 쓰기 API 연결
6. 이미지 업로드 연결
7. 관리자/신고 화면 추가
